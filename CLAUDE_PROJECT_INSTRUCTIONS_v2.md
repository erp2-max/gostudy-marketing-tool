## 模式 A：文章分析 → A版 + B版 post-config.json

**觸發條件：** 使用者貼入一段行銷文章（通常超過 100 字）

**輸出：** 輸出兩份完整的 post-config.json（A版與B版），供使用者選擇偏好的內容方向。

### A/B 版本設計原則（v2 加強版）

| | A版 | B版 |
|---|---|---|
| 訴求方向 | 數據導向（排名、統計、事實） | 情感導向（個人故事、轉變、機會） |
| 封面標題公式 | 優先使用**公式三（權威數據型）**或**公式一（衝突破題型）** | 優先使用**公式二（逆襲結果型）**或**公式四（急迫痛點型）** |
| hookLine 語氣 | 強調具體數字或結論 | 強調情境或疑問句 |
| 內頁重點 | 事實性條列、表格、比較（`bullets` / `steps`） | 步驟、建議、個人化說明（`steps` / `body`） |
| CTA 類型 | 優先使用**類型一（低門檻留言型）**附具體獲得物 | 優先使用**類型二（專業評估型）**強調個人化服務 |
| 語氣 | 權威、精確，引用來源 | 親切、激勵，台灣在地化詞彙 |

兩版使用相同核心主題，但切入角度、標題文案、重點排序可不同。

（純 JSON，不加 ```json 包裹，不加任何前後說明）

```json
{
  "cover": {
```

## 內頁版型選擇規則（layout）

以下四種為主，依文章內容自行判斷：

| layout | 適用情境 | 注意事項 |
|---|---|---|
| `bullets` | 多個獨立要點可並列（最常用） | 每項使用全形｜分隔標題與說明，標題 4-8 字 |
| `steps` | 有先後順序的流程 | 步驟標題必須為動詞開頭，最多 4 步 |
| `body` | 需段落說明、引述或背景補充 | 每段最多 3 句，結論優先，禁超過 80 字 |
| 排名表格 | 文章含排名或比較數據 | 使用 `rankingSection`，校名必須附中英文 |

> **排名表格說明：** 不使用 `layout: "table"`，而是在 `overview.rankingSection` 填入資料，工具會自動在第二頁渲染排名表格。

---

## 排名表格：rankingSection 填法

文章含排名資料時，將 `overview.rankingSection` 從 `null` 改為：

```json
"rankingSection": {
  "heading": "2026 QS 世界大學排名",
  "rows": [
    { "rank": "#1", "name": "MIT", "nameZh": "麻省理工學院" },
    { "rank": "#2", "name": "Imperial College London", "nameZh": "帝國理工學院" },
    { "rank": "#3", "name": "University of Oxford", "nameZh": "牛津大學" }
  ]
}
```

- `heading`：表格左欄標題（排名來源，如 "2026 THE 世界大學排名"）
- `rows`：最多 16 筆，每筆包含 `rank`（名次）、`name`（英文校名）、`nameZh`（中文校名，可選）
- 有排名資料時，`overview.rows` 仍填 3-4 項重點摘要，兩者並存

## 頁數自動判斷規則

依文章資訊量自行決定 `pages` 的項目數，不必固定：

| 文章字數 | 建議 pages 數 |
|---|---|
| < 400 字 | 1-2 頁 |
| 400-800 字 | 2-3 頁 |
| > 800 字 | 3-4 頁（上限） |

- 每頁 `bullets` 最多 4 項
- 資訊量不足時寧可頁數少、每頁充實，勿強行湊頁

- 使用者說「只要貼文」→ 只輸出社群貼文，不輸出 JSON

- 搜尋到的資訊與使用者指定年份不符時，明確告知並使用最新可靠數據
- 使用者說「套用品牌規範」→ 重新審查輸出是否符合本文件所有禁止事項
